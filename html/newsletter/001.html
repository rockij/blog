Vite? Next? Nuxt?

현재 투입되 진행중인 법원도서관 프로젝트는 Vite 기반의 React 프로젝트입니다.
그 이전 프로젝트는 Next 기반으로 작업한 경험이 있습니다.
여러 환경을 오가며 프로젝트를 수행하다 보니, 각각의 기술 스택이 제공하는 특성과 차이점에 대해 자연스럽게 고민하는 시간이 많아졌습니다.

먼저 각각의 환경을 한줄로 정리한다면
- Vite: “가볍고 빠르게 UI를 생산하는 프론트엔드 번들러 표준”
- Next: “SEO·라우팅·서버까지 포함한 React 풀스택 프레임워크”
- Nuxt: “Vue 생태계에서 Next 포지션을 맡는 Vue 풀스택 프레임워크”
Vite는 React, Vue 등 다양한 프레임워크에서 사용할 수 있는 ‘프런트엔드 개발 환경/번들러’ 이고, Next는 React, Nuxt는 Vue라는 점에서 차이가 있습니다.
둘다 SPA를 지원하지만 next, nuxt는 SSR(서버사이드렌더링)을 지원한다는 점에서 차이가 있습니다.

퍼블리셔 관점에서는 개발에서 환경 셋팅을 다하면 받아서 작업을 하는게 이상적이지만 프로젝트마다 성격이 달라 환경 셋팅을 직접 해야하는 경우도 있었습니다.
이런 경우라면 각각의 환경에 대해 어느정도 이해를 하고 있어야 도움이 됩니다.

작업자 입장에서 본다면 Vite는 빠른 개발환경 제공에 초점이 맞춰져 있어 npm 실행 로딩 시간이 상대적으로 빨라 작업하며 화면을 확인하는게 빠르다고 볼 수 있습니다. next, nuxt는 SEO와 라우팅, SSR등 풀스택 프레임워크로서의 역할에 초점이 맞춰져 있어 Vite 보다는 실행시 로딩이 걸리는 느낌이 있습니다.

또 Vite 기반 프로젝트는 파일 기반 자동 라우팅이 없기 때문에, 새 화면을 추가할 때 routes 설정에 직접 등록해야 했습니다. next 기반의 프로젝트는 페이지 폴더에 파일을 추가하는 것만으로 라우팅이 자동으로 되는 점이 인상적이었습니다.

작은 기준의 차이가 결과를 바꿉니다. 다음 뉴스레터에서 다시 뵙겠습니다.
