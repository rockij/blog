Next vs Vite vs Nuxt

현재 투입되 진행중인 법원도서관 프로젝트는 vite 기반의 react 프로젝트입니다.
이전 프로젝트는 next 였고 그전 프로젝트는 nuxt기반의 프로젝트 였습니다.
이런 환경에서 프로젝트를 하다보니 각각의 환경에 대해 차이점이 뭔지 고민하는 시간이 많았습니다.

먼저 각각의 환경을 한줄로 정리한다면
- Vite: “가볍고 빠르게 UI를 생산하는 프론트엔드 번들러 표준”
- Next: “SEO·라우팅·서버까지 포함한 React 풀스택 프레임워크”
- Nuxt: “Vue 생태계에서 Next 포지션을 맡는 Vue 풀스택 프레임워크”
글에서 알 수 있듯이 vite, next는 React, nuxt는 Vue라는 점에서 차이가 있습니다.
둘다 SPA를 지원하지만 next, nuxt는 SSR(서버사이드렌더링)을 지원한다는 점에서 차이가 있습니다.

퍼블리셔 관점에서는 개발에서 환경 셋팅을 다하면 받아서 작업을 하는게 이상적이지만 프로젝트마다 성격이 달라 환경 셋팅을 직접 해야하는 경우도 있었습니다.
이런 경우라면 각각의 환경에 대해 어느정도 이해를 하고 있어야 도움이 되는것을 느꼈습니다.
퍼블리셔 관점에서 본다면 vite는 빠른 개발환경 제공에 초점이 맞춰져 있고 next, nuxt는 SEO와 라우팅, SSR등 풀스택 프레임워크로서의 역할에 초점이 맞춰져 있다는 점이 가장 큰 차이점으로 느껴졌습니다.
vite, nuxt 기반의 프로젝트는 퍼블 확인 화면을 추가할 경우 별도의 파일에 코드를 추가하였지만 next 기반의 프로젝트는 페이지 폴더에 파일을 추가하는 것만으로 라우팅이 자동으로 되는 점이 인상적이었습니다.
